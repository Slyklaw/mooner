<objective>
Fix loop label namespacing bug that causes C-style for loops and while loops with break to execute incorrectly or hang infinitely.

Purpose: Labels used in WhileLoop, ForLoop, and ForInLoop handlers are not namespaced with function index, causing label collisions when multiple functions contain loops. This manifests as:
- `sum([1,2,3,4,5])` outputs 7 instead of 15 (for loop exits early)
- `while true { if condition { break } }` hangs infinitely (jump to wrong location)

Output: Fixed codegen.mbt with properly namespaced loop labels.
</objective>

<context>
@codegen.mbt
@.planning/REQUIREMENTS.md (COMP-02: Control flow constructs execute without crashes)

**Root Cause:** WhileLoop (lines 7074-7075), ForLoop (lines 7099-7101), and ForInLoop (lines 7158-7159) use bare label names like `.Lwhile_start{N}` without function index prefix. Compare to `CodeGen::new_label` at line 261 which correctly uses `.Lfn{self.current_func_idx}_{counter}`.
</context>
